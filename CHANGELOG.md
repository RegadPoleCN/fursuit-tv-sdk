## Versioning

This project uses Semantic Versioning (SemVer) for versioning. When the project reaches a stable state, versions will be released in the format `MAJOR.MINOR.PATCH`:

- **MAJOR**: Incompatible API changes
- **MINOR**: Backwards-compatible functionality additions
- **PATCH**: Backwards-compatible bug fixes

## Notes

- All dates are in YYYY-MM-DD format
- Changes are grouped by type and sorted by date
- Breaking changes are marked with **BREAKING** prefix
- Migration guides are provided for major version updates
